# Каталог `ctx/product/capabilities/`

## Назначение
Каталог `capabilities/` фиксирует ключевые продуктовые возможности, которые конкретизируют, что делает Связист в рамках обновлённой модели «Связать → звонок → ссылка по требованию». Здесь описываются состояния медиасессии, поведение кнопок, ожидания и переходы; пошаговые действия остаются в соседнем каталоге `../flows/`.

## Карта уровня
- `AGENTS.md` — текущий файл, раскрывающий назначение каталога и правила описания возможностей.
- `connection.md` — описывает мгновенный старт медиасессии после нажатия «Связать», локальное превью и placeholder удалённого потока, состояния и переходы.
- `link-sharing.md` — фиксирует кнопку «Отправить ссылку» внутри активного сеанса, выбор между Shared API и копированием, создание ссылки по требованию и UX для пожилых пользователей.

## Рекомендации для агента
- При детализации новой возможности придерживайтесь принципов доступности и минимализма; описывайте состояния, интерфейс и связи с потоками из `../flows/`.
- Используйте `../terms.md` для терминов «сеанс», «ссылка приглашения», «инициатор» и «получатель».
- Не описывайте пошаговые сценарии здесь; связывайтесь с ними через `../flows/connection.md`.
