# Каталог `ctx/product/capabilities/`

Path: `./ctx/product/capabilities/AGENTS.md`

## Назначение

Здесь описываются способности, которые превращают продукт в автономный частный центр связи: моментальный старт в одном окне, прозрачное техническое состояние и обмен ссылками внутри активной медиасессии. Документы фиксируют мотивации и соответствие архитектурным принципам, не превращаясь в пошаговые инструкции.

## Карта уровня

- `AGENTS.md` — текущий файл, объединяющий общие цели каталога и ключевые описания возможностей.
- `connection.md` — подробнее про состояния медиасессии, индикации и поддержку автономности.
- `link-sharing.md` — про генерацию ссылки внутри сессии и варианты её доставки.

## Роль документа

`AGENTS.md` выступает индексным описанием каталога: здесь слаживаются смыслы из `overview.md` и конкретные свойства, которые делают продукт автономным и воспроизводимым на любом сервере.

## Ключевые возможности

1. **Моментальный старт в одном окне.** Запуск медиасессии происходит сразу, без перехода на отдельный экран, что делает приложение доступным на любом VPS, bare-metal или контейнере и сохраняет пользовательское пространство в одном окне.
2. **Минимальное техническое состояние.** Сигнальный узел хранит только данные, необходимые для маршрутизации (WebPush-ключи, идентификаторы, ephemeral-метаданные), не создавая профилей или персональных записей, и уничтожает состояние после завершения сеанса.
3. **Обмен ссылками внутри сессии.** Генерация и передача ссылки происходят по желанию инициатора уже в активной медиасессии: Shared API и копирование доступны прямо в текущем окне без вынесенных экранов приглашения.
4. **Универсальный минимализм интерфейса.** Интерфейс остаётся понятным в любом браузере, без форм и вводов; функции видны в одном пространстве, что поддерживает доступность для любой аудитории.

## Ограничения

- Один активный звонок на сеанс, ссылка действительна только в рамках текущего соединения.
- Нет облачного хранения медиа, записей или групповых звонков; учётные записи не используются.

## Связанные документы

- `../overview.md` — смысловая основа, которой соответствуют описанные возможности.
- `connection.md` — расширенные состояния запуска, ожидания и активной передачи.
- `link-sharing.md` — детализация генерации и передачи ссылок.
- `../terms.md` — терминология (сеанс, ссылка, инициатор, получатель) для согласованности описаний.
