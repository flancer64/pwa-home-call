# Каталог `ctx/product/flows/`

## Назначение
Каталог `flows/` описывает пользовательский путь от открытия приложения до завершения медиасанса и фиксирует взаимодействие участников (`инициатор` и `получатель`) в рамках новой логики «кнопка → запуск → ссылка внутри сеанса». Здесь кратко излагается последовательность шагов, альтернативные ветки и постусловия.

## Карта уровня
- `AGENTS.md` — текущий файл с назначением каталога и рекомендациями по описанию потоков.
- `connection.md` — единственный документ уровня, описывающий поток мгновенной связи, отправку ссылки внутри сеанса и подключение получателя.

## Рекомендации для агента
- Следуй шаблону: Цель, Участники, Предусловия, Основной поток, Альтернативные потоки, Постусловия, Устаревшее поведение (если применимо).
- Перечисляй связанные возможности из `../capabilities/` и ссылку на термины из `../terms.md`.
- Избегай упоминаний отдельных экранов вне медиасессии; сценарий должен работать внутри одного окна.
