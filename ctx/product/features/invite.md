# Возможность: Приглашение по ссылке

## Назначение

Документ объясняет, как один тап превращается в ссылку приглашения, чтобы другой участник подключался без ручного выбора сеанса.

## Участники
- **Инициатор** — нажимает «Позвонить», видит экран приглашения и расшаривает ссылку.
- **Получатель** — открывает ссылку, Связист читает параметр `session`, и звонок начинается автоматически.

## Поток
1. Инициатор нажимает кнопку **Позвонить** на домашнем экране. Приложение генерирует UUID-сеанса (`sessionId`), очищает адресную строку от лишних параметров и показывает экран приглашения с видимой ссылкой `https://<домен>/?session=<uuid>`.
2. Экран приглашения всегда отображает большие кнопки **Скопировать ссылку** и **Поделиться**; если API общего доступа недоступен, видимая ссылка позволяет скопировать её вручную.
3. Карточка подчёркивает, что ссылка готова, и предлагает нажать **Начать звонок** — только после этого запускается медиа-сессия.
4. Получатель открывает ссылку, Связист получает значение `session` и подключает его автоматически; никаких форм, ошибок или подсказок о вводе данных не показываем.

## Польза
- Пожилые пользователи видят ссылку до начала звонка, поэтому понимают, куда и как её отправить.
- При невозможности поделиться через API общего доступа всегда остаётся скопировать ссылку вручную.
- Экран приглашения остаётся крупным, понятным и сохраняет путь «главный экран → приглашение → звонок». Идентификация происходит за пределами приложения, через канал доставки ссылки.
