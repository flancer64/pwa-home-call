# Продукт: Колобок

## Слоган

> Связь без границ.

## Форма

Прогрессивное веб-приложение (PWA), предназначенное для простой видеосвязи через смартфоны.

## Ценность

Колобок убирает всё лишнее: никаких регистраций, форм, запроса персональных данных и подсказок о запоминании. Сеанс создаётся автоматически и практически моментально после касания кнопки, ссылка с параметром `?session=<uuid>` появляется сама и готова к отправке.

## Пользовательский результат

Пожилой пользователь запускает приложение и видит одну крупную кнопку. Нажатие её создаёт новый сеанс, показывает ссылку, и собеседник подключается по ней без дополнительных действий. Всё происходит без сохранения данных на устройстве и без повторных запросов.

## Целевая аудитория

- пожилые люди, которым нужна минимальная навигация и максимум понятности;
- члены семей, которые хотят быстро дозвониться друг другу без технической подготовки;
- люди, ценящие автономность и отсутствие лишней информации в интерфейсе.

## Принцип

Колобок построен как **прогрессивное веб-приложение (PWA)** и опирается на технологию **WebRTC** (прямая медиасвязь), чтобы обеспечить видеообщение без облачных промежуточных сервисов. Сеансы создаются автоматически (UUID — универсальный уникальный идентификатор) при нажатии кнопки, ссылка содержит только параметр `session`, а получатель не вводит никаких данных — достаточно перейти по ссылке.

## Модель взаимодействия

- **Приложение ничего не запоминает о пользователе** и не запрашивает персональные данные.
- **Сеанс** — одноразовое соединение между инициатором и получателем, которое живёт только во время звонка и заканчивается при закрытии медиа-сессии.
- **Ссылка приглашения** — уникальный URL `https://<домен>/?session=<uuid>`, эта ссылка передаёт идентификатор сеанса.
- **Взаимодействие: одна кнопка → ссылка → звонок**. Инициатор нажимает кнопку, получает ссылку, отправляет её каналу связи, а собеседник автоматически подключается.
- **Идентификация участников** происходит за пределами приложения — через канал доставки ссылки, а не внутри интерфейса.

## Границы

- **Входит:**
  - автоматическое создание и завершение уникального сеанса;
  - однокнопочный исходящий вызов и мгновенное отображение ссылки приглашения;
  - адаптированный интерфейс для пожилых пользователей, который показывает только необходимое;
  - идентификация посредством канала доставки ссылки, без хранения персональных данных внутри Колобок.

- **Не входит:**
  - ручной ввод сеансов, имён или других персональных данных;
  - регистрация, учётные записи и облачные профили;
  - хранение медиафайлов, аналитика, многопользовательские звонки.

## Контекст

Рост потребности в заботливой и понятной связи для пожилых людей делает актуальной модель «одна кнопка → ссылка → звонок». Колобок поддерживает эту модель без лишних отвлечений и без необходимости помнить или вводить любые данные — достаточно открыть ссылку и подключиться.

## Связанные документы

- `features.md` — перечисляет ключевые возможности Колобок и связывает их с пользовательскими сценариями.
- `features/` — содержит уточняющие спецификации отдельных возможностей (например, `features/invite.md`).
- `scenarios/` — описывает пользовательские сценарии первого запуска, исходящего звонка и ежедневного использования.
- `terms.md` — содержит словарь терминов и определяет модель сеанса, ссылки приглашения, инициатора и получателя.
