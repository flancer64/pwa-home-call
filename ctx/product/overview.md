# Продукт: Связист

## Слоган
> Связь без границ.

## Форма
Прогрессивное веб-приложение (PWA) для максимально понятной видеосвязи на смартфонах.

## Ценность
Связист запускает медиасессию сразу после нажатия одной кнопки **Связать**, демонстрирует локальное видео и placeholder удалённого участника, а ссылку приглашения предлагает уже внутри активного звонка. Это исключает «экран приглашения» и избавляет пользователя от ожиданий и лишних переходов.

## Пользовательский результат
Пожилой пользователь открывает Связист, нажимает **Связать** и попадает в активный звонок с локальным превью, индикаторами камеры/микрофона и чёрным полем «Ждём собеседника». Если нужно, он нажимает **Отправить ссылку**, выбирает удобный канал или копирует URL, а собеседник сразу присоединяется в этом же окне.

## Целевая аудитория
- пожилые люди, которым важно минимум действий и максимум понятных подсказок;
- члены семьи и опекуны, которым нужны быстрые и предсказуемые звонки;
- люди, ценящие автономность, отсутствие форм и сохранения персональных данных.

## Принцип
Связист опирается на WebRTC и Web APIs платформы, чтобы запустить медиасессию моментально. UUID-сеанс создаётся единожды при нажатии **Связать**, ссылка генерируется внутри активного соединения и действует до завершения звонка.

## Модель взаимодействия
- **Приложение ничего не запоминает о пользователе** и не запрашивает личные данные.
- **Система связи:** одна кнопка **Связать** → мгновенный звонок → внутренняя кнопка **Отправить ссылку** по желанию инициатора.
- **Ссылка приглашения** создаётся по требованию внутри активного сеанса, Shared API или ручное копирование доступны из того же окна, где уже видно локальное видео и состояния.
- **Идентификация участников** происходит вне интерфейса через канал доставки ссылки, которая перестаёт работать вместе с завершением медиасессии.

## Границы
- **Входит:**
  - мгновенный запуск и завершение уникального сеанса;
  - локальное превью, placeholder удалённого потока и системные индикаторы внутри вызова;
  - отправка ссылки приглашения внутри медиасессии (Shared API + копирование);
  - минимальный интерфейс без форм и без хранения персональных данных.
- **Не входит:**
  - экран приглашения до звонка, регистрация или хранение сессий;
  - облачное хранение медиа, аналитика или групповые звонки;
  - ручной ввод идентификаторов или данных участников внутри интерфейса.

## Контекст
Рост потребности в заботливой связи формирует запрос на опыт, где всё происходит внутри одного окна: кнопка → звонок → ссылка по требованию. Связист отвечает на этот запрос, исключая отдельные экраны и предоставляя familiar experience для пожилых людей.

## Связанные документы
- `capabilities.md` — список ключевых возможностей и их взаимосвязь с новой моделью;
- `capabilities/connection.md` — детали запуска, состояний и placeholder удалённого потока;
- `capabilities/link-sharing.md` — схема кнопки «Отправить ссылку», выбор API и UX для пожилых пользователей;
- `flows/connection.md` — пошаговый поток «домашний экран → звонок → отправка ссылки → подключение → завершение»;
- `terms.md` — термины инициатора, получателя и ссылки приглашения для всей модели.
