# Сценарий: Работа с локальными данными пользователя

## Назначение

Определяет поведение ДомоЗвона при сохранении, использовании и очистке пользовательских данных, хранящихся в браузере.  
Цель — сделать повторное подключение к звонку быстрым и удобным без повторного ввода имени и комнаты.

## Участники

- **Пользователь ДомоЗвона** — человек, совершающий видеозвонки в браузере.

## Предварительные условия

- Приложение установлено и открыто в браузере.
- Пользователь ранее вводил имя и кодовое слово комнаты.

## Основной сценарий

1. Пользователь открывает ДомоЗвон впервые и вводит:
   - своё имя, под которым он будет отображаться в звонке;
   - кодовое слово (имя комнаты), к которой он подключается.
2. Приложение сохраняет эти параметры в локальном хранилище браузера.
3. При последующих запусках приложение:
   - проверяет наличие сохранённых данных;
   - если они существуют — автоматически подставляет их в форму и инициирует подключение к соответствующей комнате.
4. Пользователь при желании может изменить имя или комнату, после чего новые значения сохраняются.
5. Если сохранённые данные становятся неактуальными, пользователь может очистить локальное хранилище через интерфейс приложения.

## Альтернативные сценарии

- **A1.** При открытии без сохранённых данных приложение запрашивает ввод имени и комнаты.
- **A2.** Пользователь вручную очищает данные — приложение возвращается в состояние первого запуска.
- **A3.** При ошибке чтения локального хранилища приложение не использует данные и предлагает ввести параметры заново.

## Постусловия

- Имя пользователя и последняя активная комната сохранены в локальном хранилище.
- Приложение готово автоматически подключить пользователя при следующем входе.
- Пользователь может управлять сохранёнными данными самостоятельно.
