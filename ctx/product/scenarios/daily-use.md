# Сценарий: Повседневное использование

## Цель
Описать плавный ежедневный опыт, когда Связист открывается без подготовки и сразу работает.

## Участники
- Пожилой пользователь.
- Члены семьи или опекуны, которые подключаются по сгенерированной ссылке.

## Предусловия
- Связист установлен или закреплён, разрешения на камеру и микрофон предоставлены.

## Основной поток
1. При обычном открытии приложения главный экран показывает одну большую кнопку **Позвонить**, краткий текст о готовой ссылке и никаких полей ввода.
2. Пользователь нажимает **Позвонить**; Связист отображает экран приглашения с видимой ссылкой `https://<домен>/?session=<uuid>`, большими кнопками **Скопировать ссылку** и **Поделиться**, а также кнопкой **Начать звонок**.
3. Ссылка рассылается удобным каналом, а **Начать звонок** запускает WebRTC-сеанс, одновременно отображая индикаторы камеры и микрофона.
4. Если приложение открывается по ссылке приглашения, оно автоматически подключает пользователя к сеансу без дополнительных нажатий; экран звонка показывает статус медиа и кнопку **Повторить**, если доступ был заблокирован.

## Альтернативные потоки
- А1. API общего доступа недоступен — ссылка остаётся на экране, и пользователь копирует её вручную с помощью **Скопировать ссылку**.

## Постусловия
- Ссылка готова и доступна до начала медиа-сессии.
- Подключение по ссылке происходит без форм; любые ошибки видно через индикаторы на экране звонка.
