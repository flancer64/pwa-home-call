# Каталог `ctx/product/scenarios/`

## Назначение

Каталог `scenarios/` фиксирует пользовательские сценарии ДомоЗвон — последовательности шагов, которые выполняет конкретный участник для достижения результата.

Каждый сценарий связывает ценность из `../overview.md` и возможности из `../features.md` / `../features/` с ожидаемым опытом пользователя.

---

## Структура каталога

- `AGENTS.md` — текущий файл, карта сценариев для агента.
- `first-run.md` — сценарий первого запуска: нет сохранённого имени, ввод имени, переход к приглашению и подготовка к звонку.
- `primary-call.md` — сценарий исходящего звонка инициатора с уже сохранённым именем, создающий сеанс и приглашение за минимальное число действий.
- `daily-use.md` — сценарий повседневного использования, включая вход по ссылке приглашения и автоматическое подключение при наличии сохранённого имени.

---

## Рекомендации для агента

- При добавлении нового сценария используй формат: Purpose, Participants, Preconditions, Main flow, Alternative flows, Postconditions (как в существующих файлах, на английском языке).
- Не дублируй сюда полные описания UI-элементов — ссылайся на соответствующие возможности в `../features/*.md`.
- Проверяй согласованность сценариев со слоганом и границами из `../overview.md`: сценарии не должны вводить сущности, которых там нет (регистрация, аккаунты, облачные профили и т. п.).

## Терминология

- Перед описанием шагов заглядывай в `../terms.md` и пользуйся его определениями сеанса, ссылки приглашения, инициатора и получателя, чтобы не вводить новых сущностей.
