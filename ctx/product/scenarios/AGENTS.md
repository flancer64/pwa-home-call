# Каталог `ctx/product/scenarios/`

## Назначение

Каталог `scenarios/` фиксирует пользовательские сценарии Колобок — последовательности шагов, которые выполняет конкретный участник для достижения результата.

Каждый сценарий связывает ценность из `../overview.md` и возможности из `../features.md` / `../features/` с ожидаемым опытом пользователя.

---

## Структура каталога

- `AGENTS.md` — текущий файл, карта сценариев для агента.
- `first-run.md` — сценарий первого запуска: однокнопочный запуск с мгновенной генерацией ссылки и переходом к звонку.
- `primary-call.md` — сценарий исходящего звонка инициатора, рассказывающий о генерации ссылки, её рассылке и запуске вызова по одной кнопке.
- `daily-use.md` — сценарий повседневного использования, где приложение открывается без полей и автоматически подключает пользователя при переходе по ссылке.

---

## Рекомендации для агента

- При добавлении нового сценария используй формат: Цель, Участники, Предусловия, Основной поток, Альтернативные потоки, Постусловия (как в существующих файлах, но на русском языке).
- Не дублируй сюда полные описания UI-элементов — ссылайся на соответствующие возможности в `../features/*.md`.
- Проверяй согласованность сценариев со слоганом и границами из `../overview.md`: сценарии не должны вводить сущности, которых там нет (регистрация, аккаунты, облачные профили и т. п.).
- Все сценарии должны подчёркивать, что модель работает только через ссылку и не запрашивает от пользователя личные данные.

## Терминология

- Перед описанием шагов заглядывай в `../terms.md` и пользуйся его определениями сеанса, ссылки приглашения, инициатора и получателя, чтобы не вводить новых сущностей.
