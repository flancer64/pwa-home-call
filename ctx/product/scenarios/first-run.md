# Сценарий: Первый запуск

## Цель
Показать, что первый запуск повторяет обычный опыт — без форм и лишних шагов, только одно понятное действие.

## Участники
- Человек, который открывает ДомоЗвон впервые.

## Предусловия
- В браузере нет локальных данных, но интерфейс ничего не запрашивает.

## Основной поток
1. Домашний экран отображается крупным шрифтом, с одним понятным действием и кнопкой **Позвонить**; краткая подсказка объясняет: «Кнопка создаёт ссылку приглашения, просто поделитесь ею».
2. Пользователь нажимает **Позвонить**; ДомоЗвон создаёт новый UUID-сеанс (универсальный уникальный идентификатор), очищает URL от служебных параметров и показывает экран приглашения с видимой ссылкой `https://<домен>/?session=<uuid>` вместе с кнопками **Скопировать ссылку** и **Поделиться**.
3. Экран приглашения сохраняет крупные действия, подтверждает, что ссылка готова, и показывает **Начать звонок** — после нажатия появляется экран звонка и ждёт удалённый поток, одновременно отображая индикаторы камеры и микрофона.

## Альтернативные потоки
- А1. API общего доступа недоступен — видимая ссылка остаётся на экране, и пользователь копирует её вручную с помощью **Скопировать ссылку**.

## Постусловия
- Ссылка приглашения готова и явно доступна для отправки.
- Исходящий звонок становится активным после нажатия **Начать звонок**.
