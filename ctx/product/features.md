# Возможности продукта: Колобок

## Назначение документа

Документ фиксирует ключевые элементы поведения Колобок, фокусируясь на минималистичном опыте для пожилых пользователей.

## Ключевые возможности

### 1. Однокнопочный исходящий вызов

- Инициатор нажимает единственную кнопку «Позвонить» или аналогичную визуальную метку.
- Приложение генерирует новый UUID-сеанса (универсальный уникальный идентификатор сеанса) и очищает адресную строку от служебных параметров — пользователю не нужно вводить идентификатор вручную.
- Сеанс живёт только до окончания звонка; ссылка становится неактивной при завершении связи, а при новом вызове создаётся новый UUID (новый уникальный идентификатор).

### 2. Мгновенная генерация и рассылка ссылки приглашения

- Система строит URL `https://<домен>/?session=<uuid>` без лишних параметров и сразу предлагает его поделиться.
- На экране есть крупные кнопки «Скопировать ссылку» и «Поделиться»; при недоступности API общего доступа ссылка остаётся видимой и легко копируется вручную.
- Пользователь видит подтверждение «Ссылка готова» и не думает о настройках — достаточно выбрать удобный канал доставки.

### 3. Автоматическое подключение по ссылке

- Получатель открывает ссылку, приложение прочитывает параметр `session` и подключает его к тому же сеансу без дополнительных действий.
- Все шаги выполняются внутри приложения: ни один экран не запрашивает данные от получателя.
- Модель исключает ручной выбор сеанса или ввод каких-либо значений — только переход по ссылке.

### 4. Экран приглашения, сфокусированный на ссылке

- Карточка приглашения показывает ссылку, её статус и два действия: «Скопировать ссылку» и «Поделиться». Никаких полей для ввода.
- Кнопка «Начать звонок» появляется только после того, как ссылка готова, чтобы медиа-сессия запустилась осознанно.
- Интерфейс остаётся крупным и читабельным, подчёркивая, что пользователь уже сделал всё важное — осталось переслать ссылку.

### 5. Минималистичный интерфейс и чёткие подсказки

- Колобок скрывает технические детали, оставляя только одно действие, которое видно сразу.
- Тексты на экране — только на русском, подписывающие действия и объясняющие, что происходит с сеансом.
- Индикаторы камеры/микрофона и кнопка «Повторить» присутствуют только внутри звонка, а не до него.

### 6. Конфиденциальность без хранения данных

- Приложение не запрашивает и не сохраняет персональные данные, включая любые идентификаторы пользователей.
- Идентификация происходит вне приложения — по каналу, через который пользователь отправляет ссылку.
- Колобок хранит только техническую информацию о том, что сеанс активен, и уничтожает её после завершения.

## Ограничения текущей версии

- Один активный звонок за сессию.
- Нет групповых звонков и облачного хранения медиа.
- Нет инфраструктуры для хранения каких-либо персональных данных внутри приложения.
