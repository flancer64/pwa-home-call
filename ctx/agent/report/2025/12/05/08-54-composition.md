# Перестройка composition по контурам

Путь: `ctx/agent/report/2025/12/05/08-54-composition.md`

## Резюме изменений

- Перенёс `ctx/composition` в новую контурную структуру `client`, `signal`, `server`, `direct-interaction`.
- Вычистил достоинства архитектурных описаний, чтобы уровень фиксировал лишь артефакты; добавил свежие AGENTS и overview.
- Обновил state/ui/UX-документы, добавил messaging/ runtime-файлы и детализировал сервисные страницы.

## Детали работ

- Создал `client/screens`, `client/ui-elements`, `client/ui-states`, `client/ux` и переписал экраны, компоненты, состояния и UX-контролы с актуальными путями и ссылками.
- Вырезал документы с Flow/протоколами (`state/core-flow`, `screens/ui`, `web/...`) и вместо них добавил `signal/messaging/...`, `direct-interaction/...`, `server/service-pages/...` с уточнёнными границами.
- Переписал `ctx/composition/overview.md` и AGENTS, распространил карты по новым каталогам, чтобы структура стала навигационной.

## Результаты

- `ctx/composition` теперь содержит только описания существующих экранов, компонентов, состояний, message-форматов, runtime и сервисных страниц в рамках контуров.
- Все уровни оснащены картами: оборотни `client/`, `signal/`, `server/`, `direct-interaction/` и их подкаталоги документируют состав.
- Атрибут `read-only` для отчёта выставлен и файл отправлен вместе с другими изменениями.
