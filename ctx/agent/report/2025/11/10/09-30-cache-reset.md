# Резюме изменений
- Обновлён `CacheCleaner`: перед вызовом `location.reload()` модуль теперь удаляет все кеши и выполняет `getRegistrations().unregister()` для активных сервис-воркеров, чтобы перезапуск начинался с нуля.
- Скорректирован unit-тест и документация `ctx/rules/web/pwa.md`, чтобы покрывать поведение удаления сервис-воркера и ручной очистки кеша на экране `enter`.

# Детали работ
- Реализовано хранение `navigator` из `Env.Provider` внутри `HomeCall_Web_Pwa_CacheCleaner$` и последовательное выполнение `caches.keys()/delete`, `navigator.serviceWorker.getRegistrations()` и `location.reload()`.
- Обновлён тест `test/web/app/Pwa/CacheCleaner.test.mjs` для проверки удаления всех кешей, удаления регистрации и вызова перезагрузки.
- В документации указано, что ручная очистка теперь снимает сервис-воркер, чтобы при повторном запуске PWA кэш заново формируется с нуля.

# Результаты
- Переход «Очистить кэш» теперь запускает полную очистку: все кеши удалены, все сервис-воркеры отменены и происходит перезапуск страницы из чистой установки.
- Тест `node --test test/web/app/Pwa/CacheCleaner.test.mjs` проходит; `npm run test:web` продолжает падать из-за исторических `'test failed'` в `test/web/Core/App.test.mjs` и `test/web/Media/Manager.test.mjs`.
