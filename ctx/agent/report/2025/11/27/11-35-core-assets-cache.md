# Core assets cache update

## Резюме изменений
- Добавил в `web/service-worker.js` полный список файлов из каталога `web/` (кроме `AGENTS.md`) в `CORE_ASSETS`, чтобы все статические ресурсы предзагружались.
- Не трогал версию `web/version.json` и связанных документов не обновлял.

## Детали работ
- Сгенерировал список всех файлов внутри `web/`, исключил `AGENTS.md`, и вставил его прямо в массив `CORE_ASSETS`, убрав старые `STATIC_ASSETS`/`MODULE_ASSETS` разделы.
- Убедился, что в кэше теперь есть `service-worker.js`, `version.json`, `app/types.d.js`, дополнительные модули (`app/Config/RemoteLogging.mjs`, `app/Ui/Router/Dev.mjs`, `app/Ui/Screen/Settings.mjs`) и весь `assets/icons/` пакет.
- Проверил, что `CORE_ASSET_SET` по-прежнему используется одной модификацией во всех обработчиках сервис-воркера.

## Результаты
- `CORE_ASSETS` теперь покрывает все файлы в каталоге `web/` (без `AGENTS.md`), включая иконки и вспомогательные скрипты, поэтому кеширование статического слоя стало полным.
- Переход к единому списку упростит будущие проверки и гарантирует согласованность между реальной структурой и списком кешируемых ресурсов.
