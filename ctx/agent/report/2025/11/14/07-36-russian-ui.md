# Итерация: русская локализация UI

## Цель
- Перевести все пользовательские тексты фронта ДомоЗвон на русский (шаблоны, динамические сообщения, уведомления) и при этом сохранить англоязычные логи для разработчиков.
- Зафиксировать обновлённый языковой фокус в документации `ctx/rules/web` и поднять версию PWA, чтобы раздать новую сборку.

## Действия
- Перевёл шаблоны `web/ui/*`, `manifest.json` и служебные иконки, добавил `lang="ru"` в `index.html` и локализовал `web/app/version.json` под новую метку `20251114-073628`.
- Обновил JS-слой: русифицировал `HomeCall_Web_Ui_Screen`-компоненты, toasts и статусные строки в `HomeCall_Web_Core_App`/`HomeCall_Web_Media_Manager`, сохранил англоязычные `log.error`/`console`-выводы и русский текст для Share API.
- Привёл документацию в порядок (`ctx/rules/web/app.md`, `ui/home.md`, `ui/layout.md`, `ui/screens*.md`, `ui/share-link.md`, `contours/*`, `infra/storage.md`), зафиксировал русские строки на кнопках/подсказках и описал языковую политику.
- Запустил `npm run test:unit`; команда проходит все тесты, кроме `test/unit/Back/Service/Signal/Server.test.mjs`, который падает с `test failed`.

## Результаты
- Весь пользовательский интерфейс и toast-уведомления отображают русский текст: кнопки, сообщения о сохранении имени, статусы камеры/микрофона и сообщения о завершении звонка.
- Документация `ctx/rules/web` теперь раскрывает реальные русские заголовки и кнопки, упоминает, что логгирование остаётся на английском, и объясняет новые жесты (например, `«Позвонить»`, `«Начать звонок»`, `«Повторить»`).
- Версия PWA обновлена до `20251114-073628`, поэтому сборщик сервис-воркера сможет подхватить новую локализацию.
- `npm run test:unit` всё ещё не проходит из-за `test/unit/Back/Service/Signal/Server.test.mjs` (`'test failed'`); требуется дополнительная проверка этой зоны.
