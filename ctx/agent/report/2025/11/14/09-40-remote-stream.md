# 09-40 remote stream fix

## Резюме изменений
- добавлены гарантии, что роутер WebRTC всегда прикрепляет входящий трек даже если `event.streams` пуст или не представлен;
- повысил версию веб-слоя `web/version.json` → `20251114-093927`, чтобы фикс вошёл в следующую сборку.

## Детали работ
- расширил обработчик `track` в `web/app/Rtc/Peer.mjs`: теперь при каждом событии создан или переиспользован `MediaStream`, треки копируются из входящих потоков, а при их отсутствии корректно добавляется сам `event.track`, чтобы браузеры без `event.streams` тоже получали видео;
- добавил защиту от дублирующих добавлений и обновил `web/version.json` по UTC-таймстампу `20251114-093927`, сохранив вывод в `web/AGENTS.md`.

## Результаты
- теперь удалённый поток всегда содержит треки и передаётся на экран звонка, даже если браузер не передаёт массив `event.streams`;
- `npm run test:unit` по-прежнему падает на `test/unit/Back/Service/Signal/Server.test.mjs` с историческим сообщением "test failed"; с изменением в браузерной части новых регрессий не появилось.
