# 06-13-product-docs-for-elderly

**Цель итерации**
Переписать все документы уровня `ctx/product` так, чтобы они отражали минималистичную модель «имя один раз → инициатор нажимает кнопку → ссылка генерируется автоматически (UUID) → получатель открывает → подключение происходит само» без упоминаний кодовых слов или ручного ввода комнат.

**Действия**
- заменил `overview.md`, `features.md` и `features/invite.md` на описания модели «одинокнопочный вызов», автоматического создания ссылок и хранения только имени в локальном хранилище;
- удалил устаревшие сценарии и вместо них добавил три новых шаблона (`first-run.md`, `primary-call.md`, `daily-use.md`), которые фиксируют первый запуск, исходящий вызов и ежедневный поток в новой парадигме;
- убрал из текстов продуктового уровня все упоминания о кодовых словах, ручном выборе комнаты и старых экранах, скорректировал связи между документами, чтобы они ссылались друг на друга по новым сценариям.

**Результаты**
- продуктовый уровень теперь целиком описывает одну модель взаимодействия и ёмко отражает поведение для пожилых пользователей;
- документ `features/invite.md` и связанные `features.md`/`overview.md` согласованы с новыми сценариями и никому не позволяют вводить комнаты вручную;
- созданные сценарии показывают первый запуск, процесс создания ссылки и повседневную работу без лишних шагов.

**Соответствие критериям ADSM**
1. **Декларативность** — описано поведение (кнопка, генерация UUID, хранение имени) без деталей реализации.
2. **Полнота** — каждый сценарий раскрывает поток от первого запуска до ежедневного использования.
3. **Непротиворечивость** — один и тот же «фирменный» сценарий фигурирует во всём `product` без упоминаний прежних терминов.
4. **Связность** — `overview`, `features`, `features/invite` и сценарии образуют цепочку от ценностей к действиям.
5. **Плотность** — нет дублирующих описаний, каждый пункт в `features` объясняет новый аспект модели.
6. **Компактность** — тексты короткие, но закрывают сценарии первого запуска, инициатора и ежедневного использования.
7. **Неизбыточность** — отсутствуют старые сущности (кодовые слова, ручной ввод комнат), хранится только имя.
