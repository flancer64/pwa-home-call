# Отчёт: корректировка правил DI для тестового контейнера

## Резюме изменений
- Проанализированы причины неверного импорта и настройки контейнера `@teqfw/di` в `test/unit/helper.mjs`.
- Обновлены правила DI и тестирования в `ctx/rules/arch/teqfw/di.md` и `ctx/rules/arch/testing.md`.

## Детали работ
- Сопоставлены требования контекста с официальной документацией `@teqfw/di` и выявлена необходимость явного указания на default-импорт класса `Container`.
- Зафиксировали корректную процедуру настройки namespace через `container.getResolver().addNamespaceRoot(...)` и правило применять только задокументированные методы резолвера.

## Результаты
- Контекст дополнен точными инструкциями, что предотвращает повторение ошибочных правок в helper-модулях тестов.
- Документация синхронизирована с README пакета `@teqfw/di`, обеспечивая единообразие будущих итераций и исключая ссылки на частные примеры отсутствующих методов.
