# Header Action Button

**Path:** `./ctx/rules/web/ui/component/header-action-button.md`

## Назначение

`header-action-button` — специализированный визуальный атом для зоны `action` в `screen-header`. Он стандартизирует размеры, градиентный зелёный фон, тень и прозрачный контур, чтобы все служебные действия в заголовке выглядели как единая, читаемая точка входа для пожилых пользователей.

## API

- **Слоты**:
  - `slot="icon"` — для размещения `component/icon-wrapper` с иконкой действия.
- **Атрибуты**:
  - `aria-label` — обязательный текстовый дескриптор для обозначения назначения кнопки.
  - `disabled` — переключает внутренний `<button>` и визуально снижает контраст.

## Поведение и рекомендации

- Используется **только** внутри `screen-header` (напр., `home`, `invite`, `settings`) через слот `action`.
- События `click` всплывают через сам компонент, поэтому контроллеры привязываются к `#home-settings`, `#invite-close` и т. п. как к обычным элементам.
- Стиль определяется в компоненте и опирается на переменные `web/ui/ui.css`: `background` задаётся через `--accent-gradient`, тень наследует `--shadow-cta`, а размеры (64×64 px) и закругления повторяют общие UI-контуры (радиус 50 % чтобы кнопка была ровно круглой).
- При добавлении новых экранов с кнопкой в поле `header` достаточно поменять `<button>` на `<header-action-button>` и оставить `aria-label`/идентификаторы прежними — автоматическая стилизация и доступность сохраняются.
