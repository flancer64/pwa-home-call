# Компонент `big-button`

Path: `ctx/rules/web/ui/component/big-button.md`

## Назначение

`big-button` — универсальный CTA, используемый в зоне `action` и в служебных карточках (`settings`, `invite`, `end`). Он оформляет текст, иконку и отбивается крупными отступами без формирования логики переходов.

## API

- `label` — основной текст кнопки (используется в `action` зоне).
- `tone` (`primary | secondary | ghost`) — визуальное выделение (фон, граница).
- `disabled` — передаёт недоступность действия (только стили, никаких событий).
- `icon` — имя иконки, рендерится через `icon-wrapper`.
- `slot="secondary"` — дополнительное действие (например, «Скопировать» рядом с Share).

## Использование

- В зоне `action` `screen-card` кнопка занимает центральное пространство и получает `font-size ≥ 1.25rem`, высоту ≥ 56px и радиус ≥ 1rem.
- В `header` может появляться уменьшенный `big-button` как служебный элемент для закрытия (`tone="ghost"`, `icon="close"`), без текстовой подписи.
- Все сценарии передают в кнопку только атрибуты и не подключают обработчики: их добавляют на стороне DI-контроллера.

## Требования

- Лейбл остаётся коротким (≤ 3 слова) и не меняется динамически без нового описания экрана.
- Кнопка не содержит внутренних состояний `loading` или `hover`; состояния усиливаются атрибутами `disabled` и `tone`.
- Цвета и размеры управляются через `../style.md`; документы экранов не дублируют эти значения.

## Примечания

- `screens.md` описывает переходы, а DI-контроллеры подписываются на события кнопки (`onClick`, `onPress`); компонент остаётся чисто визуальным.
