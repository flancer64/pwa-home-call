# Компонент `screen-card`

**Path:** `ctx/rules/web/ui/component/screen-card.md`

## Назначение

`screen-card` — основной контейнер экранов. Используется всеми экранами, кроме `call`, который имеет собственный layout. Компонент обеспечивает единый каркас: ограничение ширины, вертикальную структуру `header / main / footer` и согласованность с layout. Компонент не формирует фон и не содержит логики.

> **Согласованность с продуктом:** `ctx/product/overview.md` и `ctx/product/capabilities/connection.md` говорят о наборе состояний `ready`, `waiting`, `active` внутри одного окна, поэтому `screen-card` рассматривается как фрагмент этих состояний: он не переключает экран, а лишь показывает или скрывает зоны, описываемые в `screens.md`. Будущая перестройка state machine указана в `ctx/agent/plan/2025/11/20251129-rules-implementation-fixes.md`.

## API

Компонент не имеет вариантов, статусов и динамических атрибутов.
Вся функциональность передаётся через контент слотов.

## Структура слотов

- `header` — верхняя зона.
- `main` — основная зона.
- `footer` — нижняя зона.

Назначение зон определяется экраном. Компонент гарантирует только их геометрию.

## Поведенческие требования

- Структура `header / main / footer` соответствует трёхзонной модели `layout.md`.
- Компонент не задаёт внутренних паддингов и не изменяет содержимое слотов.
- Экраны с собственным layout могут не использовать компонент.

## Требования к стилевому результату

1. **Карточка заполняет всю доступную высоту экрана.**
   Она адаптируется к высоте внешнего контейнера и не оставляет свободных зон сверху или снизу.

2. **Внутренняя структура распределена по вертикали.**

   - `header` располагается в верхней части.
   - `footer` располагается в нижней части.
   - `main` занимает всё пространство между ними.

3. **Содержимое зоны `main` центрируется внутри выделенной области.**
   Центрирование должно сохраняться при любом размере контента.

4. **Растяжение зон не нарушает ограничение ширины.**
   Карточка всегда ограничена максимальной шириной и выравнивается по центру.

5. **Компонент остаётся прозрачным.**
   Ни фон, ни декоративные слои не добавляются.
