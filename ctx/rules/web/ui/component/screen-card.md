# Компонент `screen-card`

Path: `ctx/rules/web/ui/component/screen-card.md`

## Назначение

`screen-card` — обязательная оболочка всех экранов (`screens/`). Компонент внедряет трёхзонную модель `layout.md`, задаёт максимальную ширину (≤ 640px), центрирует содержимое и оформляет фон `#002d2a → #0e6251`. Он не принимает логики: все действия, состояния и переходы приходят извне.

## API

- `data-variant` — уточняет подтон `default | elevated`, меняя тени и разделители.
- `data-status` — отражает статусы (`ready`, `waiting`, `error`), не влияя на логику, только на визуальные оттенки `screen-note`.
- `aria-live`/`aria-atomic` — передаётся для передач динамических хинтов, если `screen-note` обновляет статус.

## Структура слотов

- `slot="header"` — `screen-header`, второй уровень заголовка и вспомогательные иконки.
- `slot="action"` — главный CTA, набор `big-button` и вспомогательных действий.
- `slot="hint"` — `screen-note` или другие короткие статусы (не более 1–2 строк).

## Требования

- Оболочка привязывает зоны `header/action/hint` к трёхзонной сетке `layout.md`.
- Компонент формирует фон и ритм согласно `../style.md` и не добавляет дополнительных отступов внутри слотов.
- Все зоны заполняются сценарием: `screen-header`, `big-button`, `screen-note`, `icon-wrapper` и другие компоненты, описанные в `component/`.
- Не содержит обработчиков, внутреннего состояния или переходов; любые данные приходят из вызывающего экрана через атрибуты и слоты.

## Примечания

- `screens.md` остаётся единственным источником переходов, а `notifications.md` — статусов и ошибок.
- Экраны должны ссылаться на этот файл при описании трех зон, не дублируя параметры `layout.md` или `style.md`.
