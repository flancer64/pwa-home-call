# Компоненты UI

Path: `ctx/rules/web/ui/component/AGENTS.md`

## Назначение

Каталог `component/` содержит визуальные атомы пользовательского интерфейса Приложения. Здесь описаны компоненты, которые только оформляют DOM и принимают все данные через атрибуты и слоты; они не владеют логикой, переходами или состояниями. Экраны (`screens/`) собирают эти атомы в комбинацию, но компоненты остаются независимыми и переиспользуемыми в любой зоне `screen-card`.

## Карта уровня

- `AGENTS.md` — текущий файл, уточняющий архитектуру визуальных атомов.
- `big-button.md` — контрол для CTA, обязательный на всех экранах, с атрибутами `label`, `tone`, `disabled`, `icon`.
- `icon-wrapper.md` — универсальная обёртка для иконок, задающая контрастные фон и размеры без поведения.
- `screen-card.md` — оболочка, реализующая трёхзонную модель `header/action/hint` и задающая фон/ширину.
- `screen-header.md` — заголовок и дополнительная информация верхней зоны с поддержкой иконки и описания.
- `screen-note.md` — подсказка/статус для нижней зоны `hint`, ограниченная одной-двумя строками.

## Принципы

- Компоненты не содержат событийных или переходных обработчиков: они визуализируют значения, переданные через атрибуты, и передают действия назад только через стандартные слоты (`slot="content"`, `slot="action"` и т. п.).
- Состояние передаётся компонентам строго через атрибуты (`disabled`, `status`, `tone`, `icon` и т. д.). Никаких внутренних `useState`, `setTimeout` или DI-зависимостей внутри этих документов нет.
- Визуальный ритм и размеры задаются в `../style.md`, а трёхзонная сетка реализуется через `screen-card` и подтверждается `../layout.md`.

## Примечания

- Документы `screens/` ссылаются на компоненты через их API и подчёркивают, где `screen-header`, `screen-note` и `big-button` участвуют в каждой зоне.
- `notifications.md` остаётся единственным источником статусов, а компоненты лишь визуализируют текущие атрибуты без их изменения.
