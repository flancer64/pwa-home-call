# Базовая структура интерфейса (`ctx/rules/ui/screens.md`)

## Назначение

Документ определяет минимальный набор пользовательских экранов **HomeCall PWA**, достаточный для установления видеосвязи 1-на-1.
Цель — зафиксировать логическую основу интерфейса без визуальной детализации.

## Основные экраны

1. **Вход** — ввод кодового слова комнаты.
2. **Лобби** — отображение подключённых участников и выбор собеседника.
3. **Звонок** — активная видеосессия.
4. **Завершение** — возврат после разрыва или окончания вызова.

## Принципы

- Приложение — одноэкранное (SPA), экраны — состояния.
- Навигация линейная: Вход → Лобби → Звонок → Завершение → Лобби.
- Макет ориентирован на мобильные устройства.
- Интерфейс минимален: крупные элементы, чёткая типографика, без избыточных настроек.
- Все переходы управляются логикой приложения без перезагрузки страницы.

## Итог

Скелет HomeCall PWA образуют четыре состояния, обеспечивающие полный цикл общения: вход, выбор собеседника, звонок, завершение.
Документ фиксирует каркас пользовательского интерфейса для последующей конкретизации.
