# UI Layer

Path: `ctx/rules/web/ui/AGENTS.md`

## Назначение

Каталог `ctx/rules/web/ui/` описывает визуальные ограничения и поведение экранов Связиста. Все новые правила собираются здесь: паттерн компоновки (`layout.md`), визуальная гамма (`style.md`), toast-уведомления (`notifications.md`), шаринг ссылки (`share-link.md`), карта переходов (`screens.md`) и уникальные экраны в `screens/`.

## Карта уровня

- `screens/` — содержимое каждого экрана. Документы перечисляют только уникальные зоны, переходная карта вынесена в `screens.md`, стиль — в `style.md`.
- `AGENTS.md` — текущий файл, утверждающий карту и ссылки на правила уровня.
- `layout.md` — трёхзонная основа для `header/action/hint`.
- `notifications.md` — единая точка для toast-уведомлений и статусов.
- `screens.md` — карта экранов, CTA и переходов `home → invite → call → end`.
- `share-link.md` — механизм генерации и шаринга ссылок, fallback и тосты.
- `style.md` — визуальные ограничения (тёмная тема, крупные кнопки, системные шрифты, отсутствие hint-шума).
