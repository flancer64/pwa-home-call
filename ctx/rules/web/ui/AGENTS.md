# UI Layer

Path: `ctx/rules/web/ui/AGENTS.md`

## Назначение

Каталог `ctx/rules/web/ui/` фиксирует целевую архитектуру пользовательского слоя Связиста: два параллельных пространства — `screen/` для сценарных точек входа и `component/` для визуальных атомов. Экраны описывают уникальные зоны (`header/action/hint`), связывают DI-управляемое состояние с компонентами и не содержат логики переходов — переходы и статусы фиксируются только в `screens.md`. Компоненты оформляют DOM, принимают состояние через атрибуты и никак не управляют потоками или состояниями.

## Слои

### `screen/`

Документы внутри `screens/` (см. `screens/AGENTS.md`) описывают конкретные сценарии, уникальные элементы каждой зоны, правила взаимодействия с компонентами и целевые CTA. Экраны используют `component/screen-card.md` как обязательную оболочку и собирают `screen-header`, `screen-note` и `big-button` в нужных комбинациях. Переходы, состояния и статусы ограничиваются `screens.md` и `notifications.md`: экран лишь привязывает контент к компонентам.

### `component/`

Каталог с визуальными атомами, описанными в `component/AGENTS.md`. Компоненты не содержат логики, принимают любые значения через атрибуты и слоты, а кнопки, заголовки и заметки остаются нейтральными относительно сценариев. Все визуальные правила и ритм применяются через компоненты; экраны только наполняют их текстом и ссылками.

## Карта уровня

- `component/` — визуальные атомы, которые создают DOM `screen-card`, кнопки, заголовки и подсказки, не содержая логики (см. `component/AGENTS.md`).
- `screens/` — сценарные документы `home`, `invite`, `call`, `end`, `settings`, описывающие уникальные зоны, компоненты и переходные действия (см. `screens/AGENTS.md`).
- `AGENTS.md` — текущий файл, поясняющий архитектуру уровня.
- `layout.md` — трёхзонная сетка `header/action/hint`, реализуемая через `screen-card`.
- `notifications.md` — единственная точка для toast-уведомлений, статусов и ошибок.
- `screens.md` — карта сценариев, CTA, переходов и оставленных DI-логик.
- `share-link.md` — механизм генерации и шаринга ссылки с `toast`-подтверждениями.
- `style.md` — визуальные ограничения, применяемые к набору компонентов.
