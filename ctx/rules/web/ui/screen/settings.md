# Экран «Настройки»

**Path:** `./ctx/rules/web/ui/screen/settings.md`

## Purpose

Сервисная карточка `component/screen-card`, которая открывается из `home` через кнопку в правой части верхней зоны (`header`), и содержит `component/big-button` для служебных действий. Страница показывает только набор действий и закрытие через `component/screen-header`.

> **Согласованность с продуктом:** `ctx/product/overview.md` и `ctx/product/capabilities/connection.md` предполагают, что `settings` остаются overlay-слоем поверх текущего состояния `ready/waiting/active`, не образуя новых экранов; документ подчёркивает, что каркас `component/screen-card` используется поверх активного состояния, а `ctx/agent/plan/2025/11/20251129-rules-implementation-fixes.md` описывает, как overlay будет работать внутри единого окна.

## Zones

1. **Header** — `component/screen-header` со служебным текстом «Настройки» и иконкой закрытия.
2. **Main** — два `component/big-button` для управления удалённым логированием и переустановкой приложения; дополнительно может использоваться `icon-wrapper` для подчёркивания иконки в каждом действии.
3. **Footer** — пустая зона: подсказки не обязательны, статусы отдаются через `notifications.md`.

## Requirements

- Вся навигация происходит через `component/screen-header`, а действия описываются в DI-контроллерах, не компонентах.
- Доступ к `settings` осуществляется из `home` через кнопку в правой зоне `header`; она поднимает карточку поверх текущего экрана.
- Кнопки оформляются через `component/big-button`; экран не дублирует размеры и цвета из `style.md`.
- Закрытие сводится к возврату к экрану вызова или `home`, переходы описываются только в `screens.md`.
- Статусы и ошибки выводятся исключительно через `notifications.md`; зона `footer` пустая, `screen-note` не используется.
