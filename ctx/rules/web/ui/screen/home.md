# Home Screen

**Path:** `./ctx/rules/web/ui/screen/home.md`

## Purpose

Фиксирует запуск потока: экран содержит бренд и единственное действие, которое создаёт `sessionId` и переводит пользователя к `invite`. `component/screen-card` объединяет `component/screen-header` и `component/big-button`.

## Zones

1. **Header** — `component/screen-header` с текстом «Связист» и кнопкой настроек в правой зоне, открывающей экран `settings`.
2. **Main** — `component/big-button` **«Связать»** с иконкой, центрированная и определённая атрибутами `tone="primary"`/`icon="link"`.
3. **Footer** — `component/screen-note` появляется только для новых пользователей (одна строка «Подготовить ссылку для отправки близким»); в локации `footer` он не содержит логики и не обновляется.

## Requirements

- Нет дополнительных кнопок или текстовых полей, все элементы описаны как компоненты и управляются DI.
- Настройки открываются прямо из `home` через кнопку в правой части `header`; переход в `invite` фиксируется в `screens.md`.
- Статусы и ошибки отображаются через `notifications.md`; нижняя зона `footer` не моделирует систему состояний.
