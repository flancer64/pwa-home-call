# Home Screen

**Path:** `./ctx/rules/web/ui/screens/home.md`

## Purpose

Фиксирует запуск потока: экран содержит бренд и единственное действие, которое создаёт `sessionId` и переводит пользователя к `invite`. `component/screen-card` объединяет `component/screen-header` и `component/big-button`.

## Zones

1. **Header** — `component/screen-header` с текстом «Связист», без дополнительных иконок или доступа к настройкам.
2. **Action** — `component/big-button` **«Связать»** с иконкой, центрированная и определённая атрибутами `tone="primary"`/`icon="link"`.
3. **Hint** — `component/screen-note` появляется только для новых пользователей (одна строка «Подготовить ссылку для отправки близким»); он не используется при повторных заходах и не содержит управляемой логики.

## Requirements

- Нет дополнительных кнопок или текстовых полей, все элементы описаны как компоненты и управляются DI.
- Настройки не доступны из этого экрана, они открываются только внутри `call`; переход в `invite` фиксируется в `screens.md`.
- Статусы и ошибки отображаются через `notifications.md`; hint-зона не моделирует систему состояний.
