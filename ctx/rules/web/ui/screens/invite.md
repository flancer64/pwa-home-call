# Экран «Ссылка для соединения»

**Path:** `./ctx/rules/web/ui/screens/invite-link.md`

## Назначение

Экран выводит сформированную ссылку для приглашения другого человека в сеанс связи и предоставляет минимальный набор действий: **скопировать** или **поделиться** ссылкой (в зависимости от возможностей устройства), а также **начать звонок**.

Экран полностью перекрывает `home` как `screen-card` и возвращается обратно по нажатию FAB-кнопки закрытия в правом верхнем углу — в том же стиле, что и на экране «Настройки».

## Структура экрана

Экран состоит из трёх зон:

### 1. Зона заголовка

- Заголовок: **«Ссылка для соединения»**.
- В правом верхнем углу — FAB-кнопка **закрытия** (иконка-крестик).
- Никаких подзаголовков, поясняющих фраз или инструкций.

### 2. Зона ссылки

- Выводится только сформированная ссылка вида `https://…/?session=<uuid>`.
- Ссылка оформлена как компактный тёмный блок с мягкими углами.
- Никаких вспомогательных текстов («скопируйте», «поделитесь») — только сама ссылка.

### 3. Зона действий

Содержит **две крупные кнопки**, но набор зависит от возможностей браузера:

1. **Если доступен Web Share API (`navigator.share`)**

   - Кнопка **«Поделиться»** (иконка share).
   - Кнопка **«Начать звонок»** (иконка телефона).

2. **Если Web Share API недоступен**

   - Кнопка **«Скопировать ссылку»** (иконка copy).
   - Кнопка **«Начать звонок»** (иконка телефона).

Третья (невидимая) кнопка не должна отображаться в DOM — только те две, которые нужны для конкретного режима.

## Требования к взаимодействию

- **Нажатие «Поделиться»** использует `navigator.share`.
  Если операция недоступна или завершается ошибкой — никаких дополнительных экранов, просто fallback на «Скопировать ссылку» внутри логики приложения.
- **Нажатие «Скопировать ссылку»** копирует ссылку в буфер через Clipboard API.
- **Нажатие «Начать звонок»** переводит пользователя к экрану звонка (`call`).
- **FAB-кнопка закрытия** возвращает на домашний экран `home`.

## Требования к UI

- Минимум текста — только:

  - заголовок,
  - ссылка,
  - две кнопки действий.

- Крупные кнопки с иконками, как на других экранах.
- Основной акцентный цвет — глубокий зелёный, такой же как в `home.md`.
- Тёмная тема и градиентный фон — единый стиль приложения.
- Все элементы должны быть хорошо различимы для пожилого пользователя.

## Ограничения

- Никаких подпунктов, описаний, подсказок, инструкций.
  Экран должен быть **чрезвычайно лаконичным**.
- Только одна функция: **вывести ссылку и дать два действия**.
- Не используется ввод текста.
- Кнопки отображаются строго по условиям `share available / not available`.
