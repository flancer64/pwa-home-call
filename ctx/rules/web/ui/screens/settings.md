# Экран «Настройки»

**Path:** `./ctx/rules/web/ui/screens/settings.md`

## Purpose

Сервисная карточка `component/screen-card`, которая открывается поверх `home`/`call` и содержит `component/big-button` для служебных действий. Страница показывает только набор действий и закрытие через `component/screen-header`.

## Zones

1. **Header** — `component/screen-header` со служебным текстом «Настройки» и иконкой закрытия.
2. **Action** — два `component/big-button` для управления удалённым логированием и переустановкой приложения; дополнительно может использоваться `icon-wrapper` для подчёркивания иконки в каждом действии.
3. **Hint** — пустая зона: подсказки не обязательны, статусы отдаются через `notifications.md`.

## Requirements

- Вся навигация происходит через `component/screen-header`, а действия описываются в DI-контроллерах, не компонентах.
- Кнопки оформляются через `component/big-button`; экран не дублирует размеры и цвета из `style.md`.
- Закрытие сводится к возврату к экрану вызова или `home`, переходы описываются только в `screens.md`.
- Статусы и ошибки выводятся исключительно через `notifications.md`; `hint`-зона пустая, `screen-note` не используется.
