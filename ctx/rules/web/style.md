# Принципы визуального оформления (`ctx/rules/web/style.md`)

## Назначение

Документ фиксирует минимальные требования к внешнему виду фронтенда **HomeCall**.  
Цель — обеспечить визуальную целостность и простоту интерфейса при сохранении мобильной адаптивности.

---

## 1. Общие принципы

- Интерфейс минималистичен и функционален.
- Приоритет — **читаемость** и **доступность**, не декоративность.
- Все элементы крупные, удобные для нажатия на экране телефона.
- Цветовая схема нейтральная, с акцентом на контраст и разборчивость текста.
- Шрифты системные (`sans-serif`), без загрузки внешних гарнитур.
- Отсутствует использование фреймворков (Bootstrap, Tailwind и т.п.).
- Вёрстка основана на CSS Flexbox или Grid, без позиционирования через JS.

---

## 2. Структура стилей

| Файл               | Назначение                                                                 |
| ------------------ | -------------------------------------------------------------------------- |
| `assets/style.css` | Основные стили приложения: размеры, цвета, кнопки, типографика.            |
| `ui/*.html`        | Минимальные встроенные стили для локальных компонентов (по необходимости). |

Все стили подключаются из `index.html` одной строкой:

```html
<link rel="stylesheet" href="assets/style.css" />
```

---

## 3. Правила оформления

- Фон светлый (`#f5f5f5`), текст тёмный (`#222`).
- Акцентный цвет (например, кнопки вызова) — `#0078d7`.
- Радиус скругления — `0.5rem`.
- Используются единицы `rem` и `%`, без фиксированных пикселей.
- Минимальная высота активных зон — `44px` (по требованиям мобильной доступности).
- Анимации только CSS-транзиции, без JS.
- Все тексты интерфейса центрированы и кратки.

---

## 4. Адаптивность

- Вьюпорт фиксируется через `<meta name="viewport" content="width=device-width, initial-scale=1">`.
- Основные контейнеры масштабируются по ширине, вертикальная прокрутка разрешена.
- В портретном режиме элементы располагаются вертикально; в альбомном — горизонтально (видеопотоки).

---

## 5. Связи

- `../arch/front.md` — архитектура фронтенда.
- `./ui/screens.md` — структура экранов, на которые распространяются стили.
- `./structure.md` — размещение CSS в каталоге `assets/`.

---

## Итог

Документ задаёт минимальные визуальные стандарты HomeCall: простота, читаемость, адаптивность.
Оформление не должно отвлекать от основной функции — видеосвязи, и реализуется чистым CSS без внешних библиотек.
