# Конфигурирование логирования

**Path:** `./ctx/rules/web/logging.md`

## Назначение

Определяет способ включения удалённого логирования во фронтенд-части HomeCall.

> **Согласованность с продуктом:** `ctx/product/overview.md` указывает на минимализм UI, поэтому фронтенд-логирование хранит только технические события (сессии, состояния `ready/waiting/active`) и не содержит личной информации. Пока что интерфейс опирается на экраны `home → invite → call → end`, поэтому настройка логирования находится в настройках и отображается как переключатель; будущий single-window state machine описан в `ctx/agent/plan/2025/11/20251129-rules-implementation-fixes.md`.

## Принцип

- На экране «Настройки» размещается кнопка-переключатель **включения/выключения удалённого логирования**.
- По нажатию на эту кнопку логгер `HomeCall_Web_Logger` переводится в соответствующее состояние.
- Состояние удалённого логирования сохраняется в локальном хранилище браузера.
- Приложение при старте должно считывать это состояние и устанавливать логгер в него, строго соблюдая правила неперсонифицированных настроек (`ctx/rules/web/infra/storage.md#3`).

## Результат

Удалённое логирование включается пользователем напрямую из настроек без изменения логики логгера.
