# Стартовая страница PWA

Путь: `ctx/composition/server/service-pages/index.md`

## Назначение

`/index.html` — точка входа браузерного приложения: минимальный набор элементов и скриптов, которые позволяют `app.js` и DI-контейнеру загрузиться внутри контейнера `#app`.

## Структура страницы

- Статика включает `<div id="app">` и текстовое сообщение «Загрузка HomeCall»; пока `app.js` не замонтировал основной UI, пользователь видит эту заглушку.
- Подключён скрипт `app.js`, который собирает DI и рендерит дефолтное дерево DOM (см. `ctx/composition/client/file-structure.md`).
- Ссылка на `manifest.json`, `service-worker.js` и `version.json` держит информацию о сборке и позволяет загрузить будущие обновления.

## Поведение при неудаче

- Если `#app` остаётся пустым, появляющийся статичный блок предлагает перейти на `/reinstall.html`, чтобы пользователь мог вручную перезапустить приложение.
- Блок содержит краткую инструкцию и кнопку «Переустановить», которая ведёт к странице восстановления без перехода к другим экранам `ready/waiting/active/ended`.

## Связи

- `./reinstall.html` — ручной путь восстановления.
- `ctx/composition/client/file-structure.md` — описывает реальную структуру папок, которая загружается через эту страницу.
