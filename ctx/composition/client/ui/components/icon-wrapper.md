# **Компонент `icon-wrapper`**

Путь: `ctx/composition/client/ui/components/icon-wrapper.md`

## **Назначение**

`icon-wrapper` обеспечивает единообразное визуальное представление SVG-иконок в интерфейсе.
Он стандартизирует размер, выравнивание и отступы, чтобы все иконки выглядели согласованно и подчинялись общей теме оформления.

Компонент **не содержит поведения** и занимается только визуализацией.

---

## **API**

### **`name`**

Имя SVG-файла, расположенного в:

```text
web/assets/icons/<name>.svg
```

Компонент загружает и отображает соответствующую иконку.

### **`size`**

Стандартный размер иконки.
Используются фиксированные значения темы:

```text
small   — 24 px
medium  — 32 px
large   — 40 px
```

Эти размеры являются инвариантами визуальной системы.

---

## **Визуальные правила**

- Иконка центрируется внутри квадратной рамки, размер которой определяется параметром `size`.
- Цвет, прозрачность и контрастность полностью контролируются CSS-переменными темы приложения.
- Компонент не добавляет фон, тени или интерактивные состояния.
- `icon-wrapper` **не содержит API управления цветом**: цвет определяет контейнер, в котором он располагается.
- SVG-файл отображается без изменений: геометрия, stroke-width и viewBox остаются исходными.
- `icon-wrapper` загружает SVG через `fetch()` из `web/assets/icons/` и вставляет его как inline-SVG, что необходимо для наследования `currentColor`.
- Использование `<img>` для отображения SVG запрещено.

---

## **Использование**

`icon-wrapper` используется во всех случаях, когда интерфейс отображает иконки:

- в `big-button` — как часть CTA-действия,
- в `header-action-button` — для управляющих элементов заголовка,
- в наложениях и других компонентах UI, где требуется единообразная визуализация иконок.

Все UI-компоненты, работающие с иконками, должны использовать `icon-wrapper`,
а не вставлять SVG напрямую.
