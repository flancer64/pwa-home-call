# Декларативные UI-состояния

Путь: `ctx/composition/client/ui-states/states.md`

## Назначение

Перечисляет фрагменты `ready`, `waiting`, `active` и `ended`, указывая компоненты и наложения, присутствующие в каждом моменте, чтобы уровень композиции оставался каталогом существующих элементов, а не описанием переходов.

## Фрагмент `ready`

- `screen-card` с `screen-header`, показывающим `Связист`, и иконкой настроек в слоте действий.
- `big-button` с надписью «Связать», центрированный внутри action-зоны.
- Опциональный `screen-note` в hint-зоне, повторяющий основную задачу без введения логики состояния.

## Фрагмент `waiting`

- Наложение `share-link` (см. `ctx/composition/client/ui-states/share-link.md`) показывает URL приглашения и две `big-button` («Поделиться»/«Скопировать») над скелетом.
- `screen-header` может выводить заголовок «Ссылка для соединения» и включает кнопку закрытия в action-слоте.
- `screen-note` живёт в hint-зоне с коротким напоминанием о ссылке.

## Фрагмент `active`

- `call`-макет рендерит удалённое видео как фон, локальное превью в верхнем углу и две плавающие FAB около нижнего центра (`settings` и `hangup`).
- В этом фрагменте `screen-card` не используется; layout следует `ctx/composition/direct-interaction/media-views.md`.
- Статусы отображаются через тосты (`ctx/composition/client/ui-elements/notifications.md`), а не через hint-текст.

## Фрагмент `ended`

- `screen-card` возвращается с `screen-header`, текстом `Звонок завершён` и одной центральной `big-button` «Вернуться на главную».
- Hint-зона содержит `screen-note` с подтверждением; layout повторяет `ready`-фрагмент.

## Примечания

- Все фрагменты используют палитру и отступы из `ctx/composition/client/ui-elements/style.md` и контракты компонентов из `ctx/composition/client/ui-elements/`.
- Тосты остаются источником динамических сообщений; `screen-note` зарезервирован для статических подсказок.
