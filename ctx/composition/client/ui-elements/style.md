# Принципы визуального оформления HomeCall

Путь: `ctx/composition/client/ui-elements/style.md`

## Назначение

Фиксирует палитру, типографику, отступы, иконографику и требования к доступности, которые наследуют все компоненты и экраны. Это визуальные константы, которые делают интерфейс спокойным, читаемым и понятным для аудитории HomeCall.

## Палитра

- Базовая палитра тёмная, основана на градиенте `#002d2a → #0e6251` и сдержанном сланцевом фоне.
- Цвета текста: основной текст — `#eaf2f1`, второстепенный — `#b8c7c3`, инверсный остаётся почти чёрным для ярких слоёв обратной связи.
- Акценты: `#0e6251` для нейтральных действий, `#0faf8f` для позитивных элементов, `#6ecfc0` для информационных, `#e9a944` для предупреждений и `#d95c4a / #b04536` для ошибок. Каждому акценту соответствует контрастный цвет текста через CSS-переменные, поэтому ручная подстройка не нужна.
- Иконки используют `icon-wrapper` (см. `ctx/composition/client/ui-elements/icon-wrapper.md`) с тонами, привязанными к этой палитре: `accent` для активных глифов и `default` для пассивных.

## Типографика и масштаб

- Системные шрифты: `-apple-system`, `Segoe UI`, `sans-serif`; CTA и заголовки используют ≥ `1.25rem`, чтобы повысить читаемость.
- Основной текст — `1rem`, вспомогательный никогда не опускается ниже `0.95rem`.
- Высота `big-button` ≥ 56 px, отступы ≥ 1 rem с каждой стороны; радиусы ≥ 1 rem для мягкой, удобной формы.
- Между интерактивными элементами сохраняются отступы ≥ `0.75rem`.

## Глубина, тени и поверхности

- Фоновые поверхности (карточки, наложения) сохраняют мягкий градиент и лёгкую тень (см. `ctx/composition/client/ui-elements/notifications.md` для слоёв тостов).
- Тени всегда направлены вниз и имеют мягкий блюр; они разделяют поверхности без шума.
- `screen-card` использует эту глубину, чтобы визуально поднять действие над фоном.

## Контролы и состояние взаимодействия

- `big-button`, `header-action-button` и другие контролы получают свои визуальные состояния только из `tone` и атрибута `disabled`; никакие внутренние анимации `loading` или `hover` не описываются здесь.
- SM-контролы повторно используют палитру: `primary` — градиент, `secondary` — акцентные подсветки, `ghost` — прозрачная заливка и граница.
- Иконография остаётся монохромной, если только действие не требует сигнального цвета; допустимые значения `tone` описаны в `ctx/composition/client/ui-elements/icon-wrapper.md`.
- Состояния взаимодействия: focus — тонкий акцентный контур (`#0faf8f`); active/darkening меняют только контраст, не layout; hover-подсказки минимальны, потому что продукт ориентирован на сенсорные экраны.

## Доступность

- Широкие зоны касания (≥ 56 px) и высокий контраст обеспечивают читаемость для пожилых пользователей.
- Ширина контента ограничена 640 px на больших экранах; UI центрируется в viewport, чтобы избежать горизонтальной прокрутки.
- Длительность тостов ≥ 4 секунд и они используют `aria-live` (см. `ctx/composition/client/ui-elements/notifications.md`).

## Связи

- `ctx/composition/client/ui-elements/layout.md` — скелет, который применяет эти правила палитры.
- `ctx/composition/client/ui-elements/notifications.md` — тостовый слой, наследующий цвета, глубину и отступы.
