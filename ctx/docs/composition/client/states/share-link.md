# Состояние «Ссылка для соединения»

Путь: `ctx/docs/composition/client/states/share-link.md`

## Назначение

Фиксирует состояние `waiting`, в котором пользователь взаимодействует с URL-приглашением, но skeleton остаётся прежним, а layout определяется через overlay шаринга.

## Структура

- Наложение описано в `ctx/docs/composition/client/ui/layout/overlays/share-link.md` и располагается поверх базового `screen-card` без перестроения зон.
- Два `big-button` управляют шарингом и копированием, каждый показывает иконку через `icon-wrapper` и остаётся в едином CTA-стиле без дополнительных тонов; `screen-note` поясняет стратегию распространения ссылки.
- `share-link` остаётся видимой до тех пор, пока подключился только один WebSocket-клиент: оно показывает тот же `sessionId`, что и второй участник получает из ссылки, и подчёркивает, что контекст ожидает ещё одного пользователя без привязки к ролям.
- В этом состоянии `main-video` продолжает показывать локальное превью с `muted`, а любые наложения (включая overlay) остаются без звука, потому что вызов ещё не активен.

## Связи

- `ctx/docs/composition/client/ui/layout/overlays/share-link.md` — layout-описание overlay`а.
- `ctx/docs/composition/client/ui/patterns/toast-layer.md` — подтверждения и ошибки по шарингу всегда через тосты.
