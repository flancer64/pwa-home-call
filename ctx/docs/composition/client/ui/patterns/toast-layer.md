# Контейнер тостов HomeCall

Путь: `ctx/docs/composition/client/ui/patterns/toast-layer.md`

## Назначение

Документ фиксирует единственную область, где появляются временные статусы, подтверждения и ошибки. Тосты держат подсказки вне `screen-card` и позволяют layout’ам оставаться статичными.

## Расположение и поведение

- Контейнер закреплён у нижнего края вьюпорта, выровнен по горизонтали и отделён от зоны `action`/FAB-панели.
- Тосты используют палитру `ctx/docs/composition/client/ui/patterns/style.md` и те же правила глубины (мягкие тени, отсутствие перекрытия CTA).
- Каждое сообщение рендерится в `aria-live` регионе и может содержать иконку с тоном из сигнальных цветов, но не содержит поведения или состояния.

## Сценарии

| Сценарий              | Тип       | Стандартный текст                                       | Экранная привязка         |
| --------------------- | --------- | ------------------------------------------------------- | ------------------------- |
| Ссылка готова         | `success` | «Ссылка готова. Отправьте её собеседнику.»              | `home` / `waiting`        |
| Копирование в буфер   | `info`    | «Скопировано. Вставьте ссылку в привычный мессенджер.»  | `waiting`                 |
| Ошибка шаринга        | `warn`    | «Не удалось поделиться ссылкой.»                        | `waiting`                 |
| Разрешения медиа      | `warn`    | «Камера недоступна — можете продолжить без видео.»      | `call`                    |
| Потеря сигналинга     | `error`   | «Связь прервана. Попробуйте снова.»                     | `call`                    |
| Подключился собеседник| `info`    | «Собеседник подключён.»                                 | `call`                    |

Ни один другой элемент UI не дублирует эти сообщения; `screen-note` остаётся статичным.

## Связи

- `ctx/docs/composition/client/ui/layout/skeleton.md` — скелет оставляет место под тосты, поэтому слой размещается вне `screen-card`.
- `ctx/docs/composition/client/ui/patterns/style.md` — палитра, тени и контраст для каждого типа тоста; `ctx/docs/composition/client/ui/components/style.md` уточняет, как компоненты оформляют сообщения.
- `ctx/docs/composition/client/states/states.md` — состояния указывают, какой сценарий активен в `ready`, `waiting`, `call` и `ended`.
