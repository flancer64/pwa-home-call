# Уровень композиции HomeCall

Путь: `ctx/docs/composition/AGENTS.md`

## Назначение

Композиция организует реальные UI/UX-артефакты вокруг архитектурных контуров, не переопределяя их смыслы. Документы описывают существующие экраны, компоненты, состояния, message-форматы, runtime и сервисные страницы.

## Карта уровня

- `client/` — контур client: экраны, визуальные атомы, состояния и UX-наблюдения.
- `direct-interaction/` — визуализация p2p-среды: media-views, FAB и runtime для call.
- `server/` — сервисные точки входа (`index.html`, `reinstall.html`) и их файловая карта.
- `signal/` — фактические messaging-артефакты: форматы WebSocket и runtime сигналинга.
- `overview.md` — обзор организации композиционного уровня и его правил.
- `AGENTS.md` — текущий файл, задающий инструкцию и инварианты композиции.

## Правила для агентов

1. Категории не описывают роли или обязанности контуров — они только группируют уже существующие артефакты.
2. Документы фиксируют `Path: ...` после заголовка и выдерживают русский язык.
3. В композиции допустимы только экраны, компоненты, стили, message-форматы, runtime-детали и реальные страницы; архитектурные инварианты, environment-логика и DI-конвенции перемещаются в `ctx/docs/architecture/`/`ctx/docs/constraints/`/`ctx/docs/environment/`.
4. Карты уровня подкаталогов обеспечивают навигацию; каждая изменившаяся ветка получает собственный `AGENTS.md`.
